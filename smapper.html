The sMapper is a responsive geographical information system application designed by 3 engineering students. This application is designed for seniors who are still mobile and are experiencing aging problem. It allows seniors to perform efficient route searches and navigations while providing generic features of of a mapping software. The features in the application are designed for the needs of our intended users.

  • Implemented algorithms to solve delivery courier problem
  • Implemented path-finding functions for route searching
  • Designed user interface for seniors using Glade and GTK+

C++, Glade, GTK, Git

Grace Gan and Tiffany Yeh

The sMapper is a geographical information system application designed with C++ that supports route search for several cities around the world, such as Toronto, New York, Cairo, Beijing, Tokyo...etc. The software program provides functionality similar to Google Maps with extra features, such as quick search, hint message box, and information box. It contains an user interface that users can search route according to the location they input or location he/she clicked on the map. Besides searching fastest route between two locations, the application also supports a function that finds the fastest route for courier companies to deliver their daily deliveries.

Seniors (age 65+) who are mobile and are experiencing aging problems.

Our goal is to designed a geographical information system application that can search for the shortest route between intersections for our intended users.

While observing our grandparents, we realized that our grandparents tends to misspelled words easily because their fingers are not as agile as ours. To accomodate this problem, we implemented functions that allow our users to search route with partial street name. We included search hints on the search bar to assist our user. The search bar was designed according to our research on user interface. The research stated that by including only semantic information on search bar can make user perform better on completing the search task.

Quick search is a feature implemented to minimize the use of keyboard because keyboard is tedious and error-prone for seniors. With the quick search feature, our users will be able to obtain information of commonly visited locations (ie. restaurants, supermarkets, clinics, and subway) quickly without using keyboard.

While observing our grandparents, we realized that they tended to be frustrated when they didn't receive feedback from the program. Our solution to this problem is to implement a message box. The message box provides hint instructions and error message according to the user's actions. For detailed instructions, we also implemented a help button (see below).

There are two ways that our users can navigate the map: mouse scrolling and clicking on arrows. If the user hovers their mouse to any location on the map, the map will zoom in when scrolling. Another way is to click on the arrows indicating left, right, up, down to navigate the map.

Besides the message box, we also implemented a help button to solve the problem of having our user frustrated because of not receiving feedback from the program. The function of the help button shows detail instructions of how the should perform. For example, how to search path using street names and how quick search checklist works.

Information box provides useful route information to our users, such as travel time, points of interests, and intersections. When users clicked on intersections, the information box will show intersection name. Similarly, when users clicked on point of interests, the information box will show the name of that point (ie. Starbucks, Tim Hortons, Seven-Eleven).
